**ultrathink**

指定されたタスクに対して、以下の手順で進めてください。
各ステップでは、必要に応じて MCP サーバー `context7`（`use_context7` 等のツール）を呼び出し、プロジェクトの最新の構造や依存関係を正確に把握した上で実行してください。

1. **プロジェクトコンテキストの把握**
   - `context7` を使用して、関連するインフラ設定、データベーススキーマ、CI/CD設定を読み取る
   - 現在の実装状況を詳細に理解し、タスクの修正範囲を特定する
   - Azure リソース構成、接続文字列、環境変数の設定状況を確認する

2. **インフラ実装（azure-infra-expert）**
   - 必ず `azure-infra-expert` エージェントを起動して実装を行う
   - `context7` から得た情報を元に、既存の構成と整合性の取れた実装を行う
   - Cosmos DB、Blob Storage、App Service、CI/CDパイプラインを設定する
   - 動作確認が完了するまでこのステップを繰り返す

3. **品質検証（quality-check-expert）**
   - 必ず `quality-check-expert` エージェントを起動してレビューを行う
   - 実装内容に抜け漏れがないか、`context7` で確認できる他のファイルへの影響（破壊的変更）がないか徹底的にチェックする
   - セキュリティ設定、コスト最適化、スケーラビリティを確認する
   - 問題があればステップ 2 に戻り、すべてのレビューをクリアするまでループする

## 重要事項

- **Context7の活用**: 各ステップで必ずContext7を使用し、最新のコードベース情報を取得してください
- **品質重視**: すべてのレビュー項目がクリアされるまで、実装とレビューのループを継続してください
- **セキュリティ**: APIキー、接続文字列は必ず環境変数で管理してください
- **コスト最適化**: 不要なリソースは削除し、オートスケーリングで最適化してください
- **ドキュメント**: セットアップ手順書、インフラ構成図を作成してください
