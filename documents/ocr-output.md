## 1. 現状の課題と解決の方向性

### 現状の課題

- 既存の質疑回答生成ツールでは情報過多・文字が小さい等、役員側が使いにくい部分があった
- 事前準備における各面談内容の整理に手間を要し、想定問答集作成に時間がかかっている

### 解決の方向性

#### 株主総会支援システムの改善
- 役員にとって返答のしやすい UI・UX実現
- 文字サイズの選択、回答ボリュームの調整など

#### 事前準備サポートツール
- 音声認識で文字起こし、話者分類
- 面談記録から想定問答集の自動作成

## 2. 株主総会支援ツール

### 目的・背景

前期にご使用いただいた株主総会支援システムではオペレータ様の操作画面と役員様が回答案を確認する画面が共有されていたため、役員様にとって使われない UIや確認する回答案の文字が見えにくい事象が発生しておりました。

また、回答案が生成されるまでの時間がかかってしまう課題や質問の重要度に応じて生成 AIによる適切な回答案が出力されることが望まれます。

御社の担当者と協議した結果、以下の要件を盛り込んだ内容でまとめております。

- 役員様が返答やすい UI・UXの改善
- 役員様は生成AIの回答を直接見れるようにし、即応性を高める
- 質問の重要度によって回答レベルの選択が可能

### システム設計構想（サンプルイメージ/開発時に詳細を検討する）

役員様が閲覧するタブレットの UIが必要以上に盛り込まれる課題に対して、役員様の閲覧する画面とオペレータ様が閲覧する画面を分けることで解決することを要件として含めました。

#### 役員画面

**主要機能:**
- 文字サイズ変更機能（A+/A-ボタン）
- 質問表示エリア
- 複数の回答レベル表示（タブUI）
  - 要約（300文字）
  - 詳細（1000文字）
- スクロール機能（質問欄や解答欄がテキスト欄に収まらない場合）

**特徴:**
- 役員様にとって見やすい文字サイズに変更可能
- 質問欄や解答欄がテキスト欄に収まらない場合はテキスト欄をスクロールすることで全体を見れるようにする
- 役員様がタブUIでそれぞれの回答レベルに応じた回答を確認し状況に応じた選択することが可能
- 回答レベルの段階数は開発段階でパフォーマンスを考慮しつつ検討を進める

#### オペレータ画面

**主要機能:**

1. **音声認識機能**
   - 株主やナレータの音声を読み取り文字起こしを行う
   - 枠内のUIは一例であり、開発時に用途に合わせて調整

2. **文字数設定機能**
   - 本番の株主総会に備えて予めそれぞれの回答レベルの文字数を調整
   - 想定問答を元に適切な文字数を素早く見つけられるようになる

3. **質問入力・編集機能**
   - オペレータがメモまたは質問文を付け加えて生成AIに回答生成を要求できる
   - どのような回答を作成しますか？（入力欄）

4. **質問・回答確認エリア**
   - 質問内容表示
   - 回答表示（要約/詳細）

5. **回答モード選択**
   - チェックモード
   - ダイレクトモード

6. **回答送信機能**
   - オペレータは質問文や回答文を確認、修正したのちに役員場面に回答を送信することができる

**特徴:**
- 生成AIが要約した質問文、また質問に沿った回答文を出力
- オペレータはそれぞれの文章を確認し編集できる
- 生成AIが出力した要約、詳細の回答の結果を確認することができる

### アプリケーション概要

#### アーキテクチャ図

**構成要素:**
- **役員**: タブレット端末
- **オペレータ**: PC端末
- **Frontend**: Streamlit または React
- **リアルタイム音声認識**: Azure Speech Service
- **Backend**:
  - FastAPI
  - 生成AI
  - ベクトルDB

**注記**: 開発時に用途を検討する段階で変わる可能性があります

#### シーケンス図

**株主総会支援ツールの処理フロー:**

1. 事前に文字数を設定する（役員画面で行うかオペレータ画面で行うか後ほど検討する）
2. 文字数設定
3. 質問読み上げ
4. リアルタイム音声認識
5. 文字起こし（複数回）
6. 手動で修正/メモ入力
7. 回答生成リクエスト
8. 【ダイレクトモード】回答生成レスポンス（直接役員画面へ）
9. 【チェックモード】回答生成レスポンス（オペレータ経由）
10. 回答確認
11. 回答送信
12. 回答送信

**注記**: 開発時に用途を検討する段階で変わる可能性があります

## 4. 事前準備サポートツール

### 目的・背景

来期の株主総会に向け、役員の皆様がスムーズに回答できるよう、想定される質問に基づいた想定問答集を事前に作成いたします。

この問答集は単なる参考資料にとどまらず、株主総会支援システムのベクトルデータベースに登録され、AIによる回答生成の基盤としても活用されます。

その結果、
- 過去のやり取りや事前に整理した知見を踏まえた、精度の高い回答生成が可能になります
- 想定問答とリアルタイム生成回答を組み合わせることで、より信頼性の高いサポートが実現します

これにより、株主総会当日の負担軽減と回答品質の向上を両立させることを目指します。

### システム設計構想（サンプルイメージ/開発時に詳細を検討する）

**ワークフロー:**

1. **新規面談作成 / 面談編集**
   - 面談の作成と管理

2. **メモ入力 / リアルタイム音声認識 / 文字起こし**
   - 面談中の音声をリアルタイムで文字起こし

3. **文字起こし補正**
   - 担当者による文字起こしテキストの修正
   - 修正済み文字起こしテキストのインポート

4. **想定質問生成**
   - AIによる想定質問の自動生成

5. **想定問答生成**
   - 想定質問に対する回答の自動生成

6. **想定問答エクスポート**
   - 完成した想定問答集のエクスポート

**注記:**
- 面談時には言及されないが毎年備える想定問答などのために質問の一覧を確認・編集できるページを追加しました。

### アーキテクチャ

**処理フロー:**

1. **面談開始** （面談参加者がブラウザからアクセス）

2. **リアルタイム音声認識**
   - Azure Speech Service による音声認識

3. **文字起こし**
   - 音声データをテキスト化

4. **音声データ保存**
   - 音声DBに保存

5. **文字起こし補正リクエスト**
   - 生成AI (LLM) による自動補正

6. **文字起こし補正レスポンス**
   - 補正されたテキストを返却

7. **補正済みデータ保存**
   - 音声DBに保存

8. **補正済み文字起こしの取得**
   - 面談参加者が確認

9. **修正済み文字起こしのインポート**
   - 専門用語等により補正が難しい誤表記は面談参加者により修正される想定

10. **質問生成リクエスト**
    - Azure AIによる質問生成

11. **質問生成レスポンス**
    - 生成された質問を返却

12. **想定問答生成リクエスト**
    - Azure AIによる問答生成

13. **想定問答生成レスポンス**
    - 生成された問答を返却

14. **想定問答エクスポート**
    - 最終的な想定問答集を出力

**使用技術:**
- **Frontend**: Streamlit
- **Backend**: Azure AI Foundry
- **音声認識**: Azure Speech Service
- **データベース**: 音声DB

**注記:**
- 質問生成と想定問答生成のRAGを各々最適化した構築にすることを想定しております。今後の要件定義・検討により調整される可能性があります。
- 専門用語等により補正が難しい誤表記は面談参加者により修正される想定です。

### 生成AIを活用した想定問答生成（Q/A）の仕組み例

**処理フロー:**

1. **想定問答作成リクエスト** （ユーザーから）

2. **Web検索**
   - Web検索エンジンによる関連情報取得

3. **関連データ検索**
   - ベクトルデータベースから関連データ検索

4. **検索結果取得**

5. **関連データ検索結果評価**
   - Rerankモデル (LLM) による評価

6. **関連データ検索結果評価結果**
   - 評価結果を返却

7. **プロンプト構築**
   - 議事録 + 収集した関連情報をもとにプロンプトを構築

8. **プロンプトに基づいた想定問答生成**
   - テキスト生成モデル (LLM) による生成

9. **レスポンス** （ユーザーへ）

**使用技術:**
- Azure AI Foundry
  - Rerankモデル (LLM)
  - テキスト生成モデル (LLM)
- ベクトルデータベース
- Web検索エンジン

### ユーザ管理機能

事前準備サポートツールは複数の部署での利用を想定することで利便性が高まることが期待されます。

そのため、部署ごとに閲覧や編集権限を管理できるように、管理機能の拡張が必要になる可能性があります。

具体的には以下のような機能が将来的に必要になると思われます。
- 管理者によるユーザ管理機能（追加・削除・権限設定）
- ユーザに紐づく面談閲覧権限管理

今回の開発のスコープ外になると想定されますが、参考までに必要な開発規模感を見積もりに加えております。
