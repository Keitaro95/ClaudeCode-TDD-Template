# プロトタイプ統合テストリスト（RED）

このリストは、プロトタイプの主要な振る舞いを検証するための統合テストシナリオです。
各項目は1つのテストケースに対応し、TDDのRED-GREEN-BLUEサイクルで実装されます。

## バックエンドAPI検証

### 音声認識API
- [ ] POST /speech/recognize が固定のMock文字列を返す
- [ ] POST /speech/recognize が正しいJSON形式でtranscriptを返す

### RAG API
- [ ] POST /rag/answer が固定のMock回答（要約・詳細）を返す
- [ ] POST /rag/answer がリクエストのsummary_lengthパラメータを受け取る
- [ ] POST /rag/answer がリクエストのdetail_lengthパラメータを受け取る
- [ ] POST /rag/answer が正しいJSON形式でsummaryとdetailを返す

### ストリーミングAPI
- [ ] POST /stream/broadcast が質問・要約・詳細を受け取る
- [ ] POST /stream/broadcast が正しいレスポンス（status: "broadcasted"）を返す
- [ ] GET /stream/officer がSSEエンドポイントとして動作する
- [ ] GET /stream/officer がbroadcastされたデータをイベント形式で配信する

## オペレータ画面検証

### レイアウト
- [ ] オペレータ画面がサイドバーとメインエリアを表示する
- [ ] サイドバーに要約文字数設定が表示される
- [ ] サイドバーに詳細文字数設定が表示される

### 音声認識機能
- [ ] 音声認識開始ボタンが表示される
- [ ] 音声認識開始ボタンをクリックするとPOST /speech/recognizeを呼び出す
- [ ] 音声認識結果が質問入力エリアに自動反映される

### 回答生成機能
- [ ] 回答生成ボタンが表示される
- [ ] 回答生成ボタンをクリックするとPOST /rag/answerを呼び出す
- [ ] サイドバーの文字数設定が回答生成リクエストに含まれる
- [ ] 回答表示エリアに要約タブが表示される
- [ ] 回答表示エリアに詳細タブが表示される
- [ ] 要約タブに要約回答が表示される
- [ ] 詳細タブに詳細回答が表示される

### 回答送信機能
- [ ] 回答送信ボタンが表示される
- [ ] 回答送信ボタンをクリックするとPOST /stream/broadcastを呼び出す
- [ ] 質問・要約・詳細がすべてbroadcastリクエストに含まれる

## 役員画面検証

### レイアウト
- [ ] 役員画面に文字サイズ調整ボタン（-文字/+文字）が表示される
- [ ] 役員画面に質問表示エリアが表示される
- [ ] 役員画面に回答表示エリア（タブ付き）が表示される

### 文字サイズ調整機能
- [ ] +文字ボタンをクリックするとフォントサイズが大きくなる
- [ ] -文字ボタンをクリックするとフォントサイズが小さくなる
- [ ] フォントサイズのデフォルトが24pxである
- [ ] フォントサイズが16px未満にならない
- [ ] フォントサイズが48pxを超えない
- [ ] フォントサイズがsession_stateで管理される

### SSE接続と表示更新
- [ ] 役員画面がGET /stream/officerに接続する
- [ ] SSEで受信した質問が質問表示エリアに表示される
- [ ] SSEで受信した要約が要約タブに表示される
- [ ] SSEで受信した詳細が詳細タブに表示される
- [ ] 質問表示エリアが読み取り専用である
- [ ] 要約タブが読み取り専用である
- [ ] 詳細タブが読み取り専用である

## エンドツーエンド検証

### シナリオ1: 音声認識 → 回答生成 → 配信
- [ ] オペレータ画面で音声認識→質問入力→回答生成→送信の一連の流れが動作する
- [ ] 役員画面にオペレータからの質問が即座に表示される
- [ ] 役員画面にオペレータからの回答（要約・詳細）が即座に表示される

### シナリオ2: 文字サイズ調整
- [ ] 役員画面で文字サイズを変更すると表示が即座に更新される
- [ ] 文字サイズ変更後もSSE受信データが正しく表示される

### シナリオ3: 要約・詳細切り替え
- [ ] 役員画面で要約タブと詳細タブを切り替えられる
- [ ] タブ切り替え後も各タブに正しい内容が表示される

## 起動・アクセス検証

- [ ] バックエンドがポート8000で起動する
- [ ] Streamlitがポート8501で起動する
- [ ] オペレータ画面にhttp://localhost:8501でアクセスできる
- [ ] 役員画面にhttp://localhost:8501/officer_appでアクセスできる
